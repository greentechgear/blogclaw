## Pattern Detection Rules

### What Qualifies as a Pattern?

1. **Broken Links (3+ occurrences):**
   - Same domain fails 3+ times -> Add to domain blacklist
   - Same industry 3+ times -> Add industry watchlist

2. **Rendering Issues (3+ occurrences):**
   - Same HTML issue -> Fix in publishing scripts
   - Same markdown conversion problem -> Update converter

3. **Tone Corrections (5+ occurrences):**
   - Same phrase/word author changes -> Add to style guide
   - Same structure change -> Document in content learnings

4. **Manual Edits (5+ occurrences):**
   - Same type of edit -> Codify into reviewer
   - Same missing element -> Add to pre-publish checklist

### Pattern Action Workflow

```
Pattern Detected (3+ occurrences)
  |
  v
Analyze Root Cause
  |
  v
Determine Fix Type:
  - Script update
  - Reviewer enhancement
  - Style guide addition
  - Domain blacklist
  |
  v
Implement Fix
  |
  v
Document in SKILL_IMPROVEMENTS.md
  |
  v
Monitor for Pattern Resolution
```
